Conceptos básicos: laboratorio 1

  1. URL: Enlace de Internet para conectar a un sitio web.
  2. MIME:
  1. HTTP
  • HyperText Transfer Protocol. Principal protocolo que permite la transferencia de archivos en la Web. HTTP es textual.
  • Los errores están clasificados dentro de los siguientes rangos:
    100: Respuestas informativas.
    200: Respuestas satisfactorias.
    300: Direcciones.
    400: Errores del cliente.
    500: Errores del servidor.
  1. XML: eXtensible Markup Language. Lenguaje de marcado genérico especificado por la W3C. Permite a los usuarios definir sus propias etiquetas. Es un lenguaje de descripción de datos.
  2. XHTML: eXtensible HyperText Markup Language. Lenguaje de marcado que adapta el HTML al XHTML, volviendo a ambos compatibles.
  3. HTML5: Es una nueva versión de XHTML que permite explotar más características con menos recursos. Con este nuevo lenguaje, el estándar HTML4 deja de utilizarse, mientras que el XHTML entra en desuso al ya no ser necesario.
  4. JS: Es un lenguaje de programación que permite escribir páginas web. No requieren de compilador, pues los navegadores son quienes se encargan de interpretarlo. Es muy usado por profesionales gracias a su complejidad.
  5. PHP: HyperText Preprocessor. Es un lenguaje de programación diseñado para el desarrollo web de contenido dinámico. Está del lado del servidor gratuido e independiente de la plataforma. Posee muchas funciones y mucha documentación.
  6. JSP: Similar a PHP, crea páginas web dinámicas con base en HTML y XHTML mediante lenguaje de programación Java, lo que hace posible escribirlas en los mismos editores.
  7. AJAX: Técnica para para el desarrollo de sitios Web que implementan aplicaciones interactivas. Para hacer estas páginas se hace uso del lenguaje JavaScript.
  • ¿Cuál es la diferencia entre Internet y la World Wide Web? WWW es el sistema que se utiliza para acceder a Internet. Internet es el conjunto de redes de comunicación interconectadas que utilizan los protocolos TCP/IP.

  • ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
    GET sólo sirve para recuperar datos.
    HEAD pide una respuesta idéntica a la de HEAD, pero sin el cuerpo de la respuesta.
    POST envía una entidad a un recurso específico con el uso de formularios.
    PUT reemplaza todas las representaciones actuales del recurso description, es decir, actualiza el contenido, pero también puede crearlo.
    PATCH se usa para aplicar modificaciones parciales en un recurso. DELETE elimina un recurso identificado de una URL.

  • ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?
    POST envía mensajes en HTTP mediante formularios, por lo que es el que se requiere.

  • ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?
    GET lo hace. Mediante la URL retorna la información que tiene asociada.

  • Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?
    El código 200 es una respuesta satisfactoria que para este caso significa que el recurso HTML se ha recibido correctamente.

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?
    Los errores del rango 400 pertenecen al cliente. El 404 pertence a un recurso que no se ha encontrado. Esto significa que el cliente ha elimiado dicho recurso, lo ha movido de dirección o ha proporcionado una incorrecta. Por tanto la responsabilidad es del desarrollador.

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?
    Los errores del rango 500 son del servidor. El error 500 es una situación que no se sabe cómo manejar. El desarrollador no puede hacer nada al respecto, tiene que esperar a que el servidor pueda corregir el problema.

  • ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.
    Debido a las nuevas herramientas y características de HTML5, algunos recursos de HTML4, si bien no fueron eliminados, ya no se recomienda usarlos ya que se consideran obsoletas. Algunos ejemplos son applet, dir, isindex, menu.

  • ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?
    • HTML5 es compatible tanto con HTML4 como con XHTML.
    • El juego de caracteres "meta" se simplifica en HTML5.
    • HTML5 simpifica el DOCTYPE.
    • HTML5 incluye elementos SVG y MathML.
    • Cambio de atributos en HTML5.
    • Varios elementos como el background desaparecen del HTML4.[3]

  • ¿Cuál es el ciclo de vida de los sistemas de información?
    1. Planificación: Realizar tareas previas que influirán en la finalización exitosa del proyecto.
    2. Análisis: Averiguar lo que debe hacer el sistema. Se debe descubrir qué es lo que realmente se necesita comprendiendo los requerimientos del sistema.
    3. Diseño: Estudiar alternativas de implementación para el sistema. Se decide la estructura general del sistema.
    4. Implementación: Seleccionar herramientas, entorno que facilite el trabajo y un lenguaje de programación adecuado.
    5. Pruebas: Detectar los errores de las anteriores etapas.
    6. Instalación o despliegue: Planificar el entorno donde funcionará el sistema en software y hardware.
    7. Uso y mantenimiento: Eliminar defectos detectados en la vida útil del sistema. Adaptarlo a nuevas necesidades. Añadirle nueva funcionalidad. [4]

Preguntas: Laboratorio 2

    1. Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?
      Usarlas cuando quieres evitar que se apliquen las reglas en cascada en ciertas partes de tu documento. De este modo, la regla que se marque como !important se volverá prioritaria sobre las otras.
    2. Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?
      Las imágenes pueden hacer que el texto se oculte o sea difícil verlo.
    3. Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?
      Dependiendo tus necesidades, usa el que más te convenga. A veces, un porcentaje es más útil que un tamaño absoluto, debes decidir en qué momento es mejor cuál.
    4. ¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?
      La sangría, los espacios en blanco, comentarios y saltos de línea hacen que los archivos se vuelvan más pesados. Mientras más grande es el archivo, más se tarda en encontrar lo que requiere utilizar. Y mientras más memoria se ocupe, menor espacio ocupa en el servidor, permitiendo un mejor rendimiento.
      Eso sí, recomiendo tener copias de seguridad de los archivos CSS sin reducir para poderlos depurar cuando sea necesario.
