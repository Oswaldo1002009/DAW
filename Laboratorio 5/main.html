Preguntas (volvemos al español)

  • ¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?
    De esta manera se evita que los datos sean enviados al servidor y éste deba notificar de vuelta que el formulario no es correcto. Es mejor avisar desde javascript para que el usuario pueda corregir los inputs de inmediato y no necesitar llenarlo de nuevo.
  • ¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?
    Puede ser con inyecciones SQL, usando etiquetas en los forms de html como "validatenoform" o quitar el "onsubmit".
  • Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?
    Es un buen primer paso para después hacer las validaciones en el servidor. En cuanto a seguridad, es bueno para aprender a evitar que se inyecte código malicioso a través de inputs.
